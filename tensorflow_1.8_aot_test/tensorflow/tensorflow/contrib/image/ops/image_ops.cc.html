29REGISTER_OP("ImageProjectiveTransform")
30 .Input("images: dtype")
31 .Input("transforms: float32")
32 .Attr("dtype: {uint8, int32, int64, float32, float64}")
33 .Attr("interpolation: string")
34 .Output("transformed_images: dtype")
35 .SetShapeFn([](InferenceContext* c) {
36 c->set_output(0, c->input(0));
37 return Status::OK();
38 })
39 .Doc(R"doc(
40Applies the given transform to each of the images.
41
42Input `image` is a `Tensor` in NHWC format (where the axes are image in batch,
43rows, columns, and channels. Input `transforms` is a num_images x 8 or 1 x 8
44matrix, where each row corresponds to a 3 x 3 projective transformation matrix,
45with the last entry assumed to be 1. If there is one row, the same
46transformation will be applied to all images.
47
48If one row of `transforms` is `[a0, a1, a2, b0, b1, b2, c0, c1]`, then it maps
49the *output* point `(x, y)` to a transformed *input* point
50`(x', y') = ((a0 x + a1 y + a2) / k, (b0 x + b1 y + b2) / k)`, where
51`k = c0 x + c1 y + 1`. If the transformed point lays outside of the input
52image, the output pixel is set to 0. The output is the same size as the input,
53
54images: 4D `Tensor`, input image(s) in NHWC format.
55transforms: 2D `Tensor`, projective transform(s) to apply to the image(s).
56
57transformed_images: 4D `Tensor`, image(s) in NHWC format, generated by applying
58the `transforms` to the `images`. Satisfies the description above.
59)doc");

61REGISTER_OP("BipartiteMatch")
62 .Input("distance_mat: float")
63 .Input("num_valid_rows: float")
64 .Attr("top_k: int = -1")
65 .Output("row_to_col_match_indices: int32")
66 .Output("col_to_row_match_indices: int32")
67 .SetIsStateful()
68 .SetShapeFn([](InferenceContext* c) {
69 ShapeHandle input;
70 TF_RETURN_IF_ERROR(c->WithRank(c->input(0), 2, &input));
71 c->set_output(0, c->MakeShape({c->Dim(input, 0)}));
72 c->set_output(1, c->MakeShape({c->Dim(input, 1)}));
73 return Status::OK();
74 })
75 .Doc(R"doc(
76Find bipartite matching based on a given distance matrix.
77
78A greedy bi-partite matching algorithm is used to obtain the matching with the
79(greedy) minimum distance.
80
81distance_mat: A 2-D float tensor of shape `[num_rows, num_columns]`. It is a
82 pair-wise distance matrix between the entities represented by each row and
83 each column. It is an asymmetric matrix. The smaller the distance is, the more
84 similar the pairs are. The bipartite matching is to minimize the distances.
85num_valid_rows: A scalar or a 1-D tensor with one element describing the
86 number of valid rows of distance_mat to consider for the bipartite matching.
87 If set to be negative, then all rows from `distance_mat` are used.
88top_k: A scalar that specifies the number of top-k matches to retrieve.
89 If set to be negative, then is set according to the maximum number of
90 matches from `distance_mat`.
91row_to_col_match_indices: A vector of length num_rows, which is the number of
92 rows of the input `distance_matrix`.
93 If `row_to_col_match_indices[i]` is not -1, row i is matched to column
94 `row_to_col_match_indices[i]`.
95col_to_row_match_indices: A vector of length num_columns, which is the number
96 of columns of the input ditance matrix.
97 If `col_to_row_match_indices[j]` is not -1, column j is matched to row
98 `col_to_row_match_indices[j]`.
99)doc");

101REGISTER_OP("ImageConnectedComponents")
102 .Input("image: dtype")
103 .Output("components: int64")
104 .Attr(
105 "dtype: {int64, int32, uint16, int16, uint8, int8, half, float, "
106 "double, bool, string}")
107 .SetShapeFn([](InferenceContext* c) {
108 return shape_inference::UnchangedShape(c);
109 })
110 .Doc(R"doc(
111Find the connected components of image(s).
112
113For each image (along the 0th axis), all connected components of adjacent pixels
114with the same non-zero value are detected and given unique ids.
115
116The returned `components` tensor has 0s for the zero pixels of `images`, and
117arbitrary nonzero ids for the connected components of nonzero values. Ids are
118unique across all of the images, and are in row-major order by the first pixel
119in the component.
120
121Uses union-find with union by rank but not path compression, giving a runtime of
122`O(n log n)`. See:
123 https://en.wikipedia.org/wiki/Disjoint-set_data_structure#Time_Complexity
124
125image: Image(s) with shape (N, H, W).
126components: Component ids for each pixel in "image". Same shape as "image". Zero
127 pixels all have an output of 0, and all components of adjacent pixels with
128 the same value are given consecutive ids, starting from 1.
129)doc");
