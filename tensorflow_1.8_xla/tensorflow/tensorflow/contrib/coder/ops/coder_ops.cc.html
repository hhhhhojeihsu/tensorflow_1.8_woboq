25// clang-format off
26REGISTER_OP("RangeEncode")
27 .Input("data: int16")
28 .Input("cdf: int32")
29 .Output("encoded: string")
30 .Attr("precision: int >= 1")
31 .SetShapeFn(shape_inference::ScalarShape)
32 .Doc(R"doc(
33Using the provided cumulative distribution functions (CDF) inside `cdf`, returns
34a range-code of `data`.
35
36The shape of `cdf` should have one more axis than the shape of `data`, and the
37prefix `cdf.shape[:-1]` should be broadcastable to `data.shape`. That is, for
38every `i = 0,...,rank(data) - 1`, the op requires that either
39`cdf.shape[i] == 1` or `cdf.shape[i] == data.shape[i]`. Note that this
40broadcasting is limited in the sense that the number of axes must match, and
41broadcasts only `cdf` but not `data`.
42
43`data` should have an upper bound `m > 0` such that each element is an integer
44in range `[0, m)`. Then the last dimension size of `cdf` must be `m + 1`. For
45each element of `data`, the innermost strip of `cdf` is a vector representing a
46CDF. For each k = 0,...,m, `cdf[..., k] / 2^precision` is the probability that
47an outcome is less than `k` (not less than or equal to).
48
49```
50 cdf[..., 0] / 2^precision = Pr(data[...] < 0)
51 cdf[..., 1] / 2^precision = Pr(data[...] < 1) = Pr(data[...] <= 0)
52 cdf[..., 2] / 2^precision = Pr(data[...] < 2) = Pr(data[...] <= 1)
53 ...
54 cdf[..., m] / 2^precision = Pr(data[...] < m) = 1
55```
56
57Therefore each element of `cdf` must be in `[0, 2^precision]`.
58
59Ideally `cdf[..., m]` should equal to `2^precision` but this is not a hard
60requirement as long as `cdf[..., m] <= 2^precision`.
61
62The encoded string neither contains the shape information of the encoded data
63nor a termination symbol. Therefore the shape of the encoded data must be
64explicitly provided to the decoder.
65
66Implementation notes:
67
68- Because of potential performance issues, the op does not check whether
69elements of `data` is in the correct range `[0, m)`, or if `cdf` satisfies
70monotonic increase property.
71
72- For the range coder to decode the encoded string correctly, the decoder should
73be able to reproduce the internal states of the encoder precisely. Otherwise,
74the decoding would fail and once an error occur, all subsequent decoded values
75are incorrect. For this reason, the range coder uses integer arithmetics and
76avoids using any floating point operations internally, and `cdf` should contain
77integers representing quantized probability mass rather than floating points.
78
79data: An int32 tensor.
80cdf: An int32 tensor representing the CDF's of `data`. Each integer is divided
81 by `2^precision` to represent a fraction.
82encoded: A range-coded scalar string.
83precision: The number of bits for probability quantization. Must be <= 16.
84)doc");

87REGISTER_OP("RangeDecode")
88 .Input("encoded: string")
89 .Input("shape: int32")
90 .Input("cdf: int32")
91 .Output("decoded: int16")
92 .Attr("precision: int >= 1")
93 .SetShapeFn([] (InferenceContext* c) {
94 ShapeHandle out;
95 TF_RETURN_IF_ERROR(c->MakeShapeFromShapeTensor(1, &out));
96 c->set_output(0, out);
97 return Status::OK();
98 })
99 .Doc(R"doc(
100Decodes a range-coded `code` into an int32 tensor of shape `shape`.
101
102This is the reverse op of RangeEncode. The shape of the tensor that was encoded
103should be known by the caller.
104
105Implementation notes:
106
107- If wrong input was given (e.g., corrupt `encoded` string, or `cdf` or
108`precision` do not match encoder), the decode is unsuccessful. Because of
109potential performance issues, the decoder does not return error status.
110
111encoded: A scalar string tensor from RangeEncode.
112shape: An int32 1-D tensor representing the shape of the data encoded by
113 RangeEncode.
114decoded: An int32 tensor with shape equal to `shape`.
115precision: The number of bits for probability quantization. Must be <= 16, and
116 must match the precision used by RangeEncode that produced `encoded`.
117)doc");
