24// TPUs use a specialized mechanism for performing embedding lookups,
25// necessitating differences in TF Graphs that use embeddings on TPUs relative
26// to CPUs. Embedding lookups on TPU systems are achieved by including the
27// following in the TF Graph.
28//
29// 0. Construct a TPUEmbeddingConfiguration, specifying the embedding tables
30// in the model, the size of the TPU system to be used, and the optimizer to
31// be used for each table. Some of this information is redundant with other
32// pieces of the TF Graph.
33// 1. Pass this TPUEmbeddingConfiguration to tpu.initialize_system() as the
34// tpu_embedding_config parameter.
35// 2. Use the TPUEmbeddingLoad Op to initialize the embedding tables in TPU
36// memories, sharded across the memories attached to each Host.
37// 3. Use TPUEmbeddingEnqueueSparseBatch to provide the TPU with embedding
38// indices and aggregation weights.
39// 4. TPUEmbeddingReceiveActivations returns a list of Tensors, containing the
40// activations from each table specified in the configuration.
41// 5. TPUEmbeddingActivations, when used with appropriate Python libraries,
42// enables the automatic differentiation of models that use embeddings.
43// 6. TPUEmbeddingSendGradients takes a list of Tensors (of the same shapes
44// as those returned by TPUEmbeddingReceivActivations) containing gradients
45// to use in updating the embedding tables.
46// 7. Before saving a checkpoint, use the TPUEmbeddingRetrieve Op to update
47// the Graph's embedding table Variables from the updated tables in the
48// TPU memories.
49//
50// TPU Embeddings use dedicated ops to enforce Host/TPU consistency in the
51// state of embedding table variables. Before beginning training or inference,
52// the model must Load the optimizer parameters into the TPU memories. Before
53// saving a checkpoint, the model must Retrieve the parameters back into the
54// host CPU memory.
55
56REGISTER_OP("TPUEmbeddingLoadGradientDescentParameters")
57 .Input("parameters: float32")
58 .Attr("tpu_embedding_config: string")
59 .Attr("table_id: int >= 0")
60 .Attr("num_hosts: int >= 1")
61 .Attr("host_id: int >= 0")
62 .SetIsStateful()
63 .SetShapeFn(shape_inference::UnknownShape)
64 .Doc(R"doc(
65Load an embedding table shard into TPU memory for use with GradientDescent.
66
67TPU embeddings use dedicated per-optimizer Ops for loading and retrieving
68trainable variables and optimizer state from TPU memory. This op enables
69functionality equivalent to GradientDescentOptimizer.
70
71parameters: The shard of the embedding table resident on the host executing this
72 op. For single-TPU models, this is the entire embedding table.
73tpu_embedding_config: Serialized TPUEmbeddingConfiguration proto.
74table_id: The id of the table specified in the tpu_embedding_config.
75num_hosts: The number of CPU hosts in the distributed training job.
76host_id: Which CPU host in the distributed training job will execute this op.
77)doc");

79namespace tpu_embedding_config_util {

81Status GradientDescentShapes(shape_inference::InferenceContext *c) {
82 string config_string;
83 TF_RETURN_IF_ERROR(c->GetAttr("tpu_embedding_config", &config_string));
84 tpu::TPUEmbeddingConfiguration config;
85 if (!config.ParseFromString(config_string)) {
86 return errors::InvalidArgument("Malformed tpu_embedding_config.");
87 }
88
89 int table_id;
90 TF_RETURN_IF_ERROR(c->GetAttr("table_id", &table_id));
91 int64 num_tables = config.table_config_size();
92 if (table_id >= num_tables) {
93 return errors::InvalidArgument("Table id >= num_tables");
94 }
95 int64 width = config.table_config(table_id).width();
96 int64 num_rows = config.table_config(table_id).num_rows();
97
98 TF_RETURN_IF_ERROR(c->set_output("parameters", {c->Matrix(num_rows, width)}));
99 return Status::OK();
100}

102} // namespace tpu_embedding_config_util

200REGISTER_OP("TPUEmbeddingEnqueueSparseBatch")
201 .Input("sample_indices: num_tables * int32")
202 .Input("embedding_indices: num_tables * int32")
203 .Input("aggregation_weights: num_tables * float32")
204 .Attr("num_tables: int")
205 .Attr("device_ordinal: int = -1")
206 .SetIsStateful()
207 .SetShapeFn(shape_inference::UnknownShape)
208 .Doc(R"doc(
209An op that feeds a batch of embedding indices and weights to the TPU.
210
211Embedding lookups are equivalent to sparse-dense matrix multiplications: the
212sparse matrix contains nonzeros in column j in order to retrieve row j from the
213embedding table.
214
215The three Tensor list arguments (sample_indices, embedding_indices, and
216aggregation_weights) represent these sparse matrices in COO format. The Tensor
217lists each have one entry for each embedding table specified in the model.
218For the kth embedding table, the three Tensors at position k in the list
219specify a COO-format sparse matrix. For the kth table, the row indices,
220column indices, and nonzero values of the COO sparse matrix are specified by
221sample_indices[k], embedding_indices[k], and aggregation_weights[k],
222respectively. Entries must be sorted by row index, then by column index.
223
224There should be at most one TPUEmbeddingEnqueueSparseBatch op in a signle
225training step per TPU shard.
226
227sample_indices: A list of rank 1 Tensors specifying row indices of the COO
228 sparse matrix representing the embedding lookups for each table.
229embedding_indices: A list of rank 1 Tensors specifying column indices of the
230 COO sparse matrix representing the embedding lookups for each table.
231aggregation_weights: A list of rank 1 Tensors specifying the nonzero values
232 of the COO sparse matrix representing the embedding lookups for each table.
233device_ordinal: The TPU device to use. This should be -1 when the Op
234 is running on a TPU device, and >= 0 when the Op is running on the CPU
235 device.
236)doc");

240Status ActivationShapes(shape_inference::InferenceContext *c) {
241 string config_string;
242 TF_RETURN_IF_ERROR(c->GetAttr("tpu_embedding_config", &config_string));
243 tpu::TPUEmbeddingConfiguration config;
244 if (!config.ParseFromString(config_string)) {
245 return errors::InvalidArgument("Malformed tpu_embedding_config.");
246 }
247 int64 batch_size = config.batch_size();
248 int64 num_tables = config.table_config_size();
249 for (int table_id = 0; table_id < num_tables; ++table_id) {
250 int64 width = config.table_config(table_id).width();
251 int64 num_features = config.table_config(table_id).num_features();
252 c->set_output(table_id, c->Matrix(batch_size * num_features, width));
253 }
254 return Status::OK();
255}

257} // namespace tpu_embedding_config_util

259REGISTER_OP("TPUEmbeddingReceiveActivations")
260 .Output("outputs: num_tables * float")
261 .Attr("num_tables: int >= 1")
262 .Attr("tpu_embedding_config: string")
263 .SetIsStateful()
264 .SetShapeFn(tpu_embedding_config_util::ActivationShapes)
265 .Doc(R"doc(
266An op that receives embedding activations on the TPU.
267
268The TPU system performs the embedding lookups and aggregations specified by
269the arguments to TPUEmbeddingEnqueueSparseBatch. The results of these
270aggregations are visible to the Tensorflow Graph as the outputs of a
271TPUEmbeddingDequeueActivations Op. This op returns a list containing one
272Tensor of activations per table specified in the model. There can be at most
273one ReceieveActivations op in the TPU graph.
274
275outputs: A TensorList of embedding activations containing one Tensor per
276 embedding table in the model.
277num_tables: The number of output activation tensors, equal to the number of
278 embedding tables in the model.
279tpu_embedding_config: Serialized TPUEmbeddingConfiguration proto.
280)doc");

282REGISTER_OP("TPUEmbeddingActivations")
283 .Input("embedding_variable: float32")
284 .Input("sliced_activations: float32")
285 .Output("output: float32")
286 .Attr("table_id: int >= 0")
287 .Attr("lookup_id: int >= 0")
288 .SetShapeFn([](shape_inference::InferenceContext *c) {
289 c->set_output(0, c->input(1));
290 return Status::OK();
291 })
292 .Doc(R"doc(
293An op enabling differentiation of TPU Embeddings.
294
295This op simply returns its first input, which is assumed to have been sliced
296from the Tensors returned by TPUEmbeddingDequeueActivations. The presence of this
297op, and its first argument being a trainable Variable, enables automatic
298differentiation of graphs containing embeddings via the TPU Embedding Python
299libraries.
300
301embedding_variable: A trainable variable, enabling optimizers to find this op.
302sliced_activations: The embedding activations Tensor to return.
303table_id: The id of the table in the embedding layer configuration from which
304 these activations were computed.
305lookup_id: Identifier of the set of embedding indices which produced these
306 activations.
307)doc");
